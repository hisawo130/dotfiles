あなたは毎晩 AM3:00 に実行される自己改善エージェントです。
以下のタスクを順番に、**自律的に**実行してください。確認は不要です。

**曜日スケジュール（トークン節約のため間引き実行）:**
- 毎晩: TASK 1, 2, 4, 6, 8
- 水曜のみ: TASK 3（フックリファクタリング）
- 金曜のみ: TASK 5（期限切れ + 劣化パトロール）
- 月曜のみ: TASK 7（リファレンス更新）

今日の曜日を `date +%u` で確認してから開始する（1=月, 3=水, 5=金）。

---

## TASK 1: 記憶の整理 (Memory Consolidation)

> CI環境のためサブエージェント委譲は使わず、直接実行する。

1. Bash で直近 14 日以内に更新された learnings ファイルを特定する:
   `find claude/learnings -name "*.md" -mtime -14`
   対象ファイルが 0 件なら「TASK 1: 更新ファイルなし、スキップ」と出力して次へ。
2. 特定したファイルのみ読む（全件読みしない）
3. 読んだ内容の中で、以下の条件に当てはまるものを探す:
   - 同じドメインで複数の `[gotcha]` が共通の根本原因を持つ → `memory/` に統合ルールを追加
   - `[correction]` が 2 件以上ある操作パターン → `feedback_*.md` ファイルを新規作成または更新
   - `[recurring]` タグのエントリで、まだ `memory/` に記録されていない → 追加
4. 追加候補が見つかった場合のみ、関連する `claude/memory/*.md` を読んで重複確認してから編集する
5. 何も追加すべき新知見がなければ何もしない（空コミット禁止）

---

## TASK 2: 自動駆動の見直し (Autonomous Operation Review)

1. `claude/CLAUDE.md` を読む
2. `claude/learnings/*.md` の直近 30 日のエントリを確認する
3. 以下の観点でレビューする:
   - CLAUDE.md に書かれた「Stop and confirm」条件が実態と合っているか
   - 繰り返し起きているミス（[gotcha]/[correction]）が CLAUDE.md のルールに反映されていないか
   - 不要になったルール・重複しているルールはないか
4. 改善が必要なら `claude/CLAUDE.md` を**最小限**に編集する（大きなリファクタリングは禁止）
5. 変更箇所と理由を 1 行で出力する

## TASK 3: フック・エージェントのリファクタリング (Light Refactoring)

> **水曜日のみ実行。** それ以外の曜日は「TASK 3: スキップ（曜日制限）」と出力して次へ。

1. `claude/hooks/*.sh` と `claude/agents/*.md` をざっと確認する
2. 以下の軽微な修正のみ実施する（大規模変更は禁止）:
   - シェルスクリプトの明らかなバグ（`fi` / `done` / `esac` の閉じ忘れ等）
   - コメントが実装と乖離している箇所
   - 同じ処理が 3 箇所以上コピーされていて、共通ライブラリに移動できるもの
3. 変更した場合は何を直したかを 1 行で出力する
4. 問題がなければ「リファクタリング不要」と出力してスキップする

## TASK 4: 自己成長ログの生成 (Growth Log)

以下の内容を `claude/scripts/growth-log.md` に**追記**する（上書き禁止）:

```markdown
## {YYYY-MM-DD} 夜間自己改善レポート

### タスク実行サマリー
- TASK 1: {変更あり N件 / 変更なし / スキップ（更新ファイルなし）}
- TASK 2: {変更あり / 変更なし}
- TASK 3: {変更あり / 変更なし / スキップ（曜日制限）}
- TASK 4: 完了
- TASK 5: {変更あり N件 / 変更なし / スキップ（曜日制限）}
- TASK 6: 完了
- TASK 7: {更新あり / 変更なし / スキップ（曜日制限）}
- TASK 8: 完了

### 記憶整理
- 追加したメモリルール: {件数} 件
- 内容: {追加した内容の要約、なければ「なし」}

### 自動駆動の見直し
- CLAUDE.md の変更: {変更箇所の説明、なければ「なし」}

### 今日の観察
{学習ログから読み取れる傾向・パターン・注意点を 1〜2 文で簡潔に}
```

**growth-log.md の肥大化防止（追記後に実行）:**
追記が終わったら `claude/scripts/growth-log.md` の行数を確認する。
90 日以前（今日から 90 日より前の日付）の `## YYYY-MM-DD` セクションが存在する場合:
- 該当月のエントリを 1 行のサマリー `## YYYY-MM 月次まとめ（{件数}日分を圧縮）` に置き換える
- 月次まとめが既にあればスキップ
- 圧縮後にファイルを上書き保存する

## TASK 5: 期限切れデータ + 劣化エントリのパトロール (Stale Patrol)

> **金曜日のみ実行。** それ以外の曜日は「TASK 5: スキップ（曜日制限）」と出力して次へ。

### 5-A: 期限切れデータ
1. `claude/memory/*.md` を読む
2. 過去日付が「最優先」「要対応」「⚠️」として残っている場合 → 「✅ 期限済み」に書き換える
3. 変更がなければ「期限切れデータなし」と出力

### 5-B: 劣化エントリの除去
1. `claude/learnings/*.md` を読む
2. 以下の条件を**両方**満たすエントリを特定する:
   - セクション日付（`## YYYY-MM-DD`）が今日から **180日以上前**
   - `[gotcha]` / `[tip]` / `[pattern]` / `[correction]` / `[open]` のいずれのタグも含まない（= タグなし作業ログ）
3. 該当セクションを丸ごと削除する（`[recurring]` セクションは触らない）
4. 削除したセクション数と対象ファイルを 1 行で出力する
5. 該当なければ「劣化エントリなし」と出力

## TASK 6: 学習メトリクスの記録 (Learning Metrics)

1. `claude/learnings/` 配下の全ドメインファイルについて、Bash で行数を確認する:
   `wc -l claude/learnings/*.md`
   3 行以下（ヘッダーのみ）のファイルは「未蓄積」としてリストに記録するだけでよく、Read で読まない。
2. 4 行以上のファイルのみ Read で読み、以下を集計する:
   - 各ドメインのエントリ総数
   - `[gotcha]` / `[recurring]` / `[pattern]` / `[correction]` / `[open]` の件数
   - エントリが 0 件（空）のドメイン一覧
2. 集計結果を以下のフォーマットで `claude/scripts/growth-log.md` の当日レポートに追記する:

```markdown
### メトリクス
| ドメイン | 合計 | gotcha | recurring | 最終更新 |
|---|---|---|---|---|
| shopify | 5 | 3 | 1 | 2026-03-27 |
...

未蓄積ドメイン: cloudflare, cms, ec-cube, ...
```

## TASK 7: リファレンスの週次更新 (Weekly Reference Refresh)

> このタスクは **週1回のみ実行**する。今日が月曜日でない場合はスキップ。

1. `~/.claude/references/` 配下の全 `.md` ファイルを確認する
2. `UPDATE BEFORE USE` ブロックを含むファイルのみ対象とする
3. 各ファイルの `Sources:` リストから URL を抽出し、WebFetch で取得する
4. 現在のファイル内容と比較し、以下があれば更新する:
   - 新しいAPIバージョン・新機能・廃止された機能・変更された制限値
   - リリースノートや変更ログの新しいエントリ
   - 既存の内容は削除しない（明確に廃止されたものを除く）
5. 更新があったファイルを書き込む（必ず完全なファイル内容で上書き）
6. 更新があれば以下を実行する（別々の呼び出しで）:
   - `git -C ~/dotfiles add claude/references/`
   - `git -C ~/dotfiles commit -m "docs: リファレンス週次更新 $(date +%Y-%m-%d)"`
   - `git -C ~/dotfiles push`
7. 以下フォーマットで growth-log.md の当日レポートに追記する:

```markdown
### リファレンス更新
| ファイル | ステータス | 変更箇所 |
|---|---|---|
| shopify-reference.md | ✅ 変更なし / 🔄 更新済み | - |
...
```

## TASK 8: 自己効率化リフレクション (Self-Efficiency Reflection)

> 毎晩実行。全タスク完了後に最後に行う。

1. `claude/scripts/growth-log.md` から**直近30日分**のレポートを読む
2. 各タスク（TASK 1〜7）について以下を集計する:
   - 「変更あり」「変更なし/スキップ」「曜日制限スキップ」の回数
3. 以下の条件に当てはまるタスクを特定する:
   - **4回以上連続**で「変更なし」または「更新ファイルなし、スキップ」だった → 過剰実行の可能性
   - **毎回変更あり**で処理量が多い → 分割や対象絞り込みの余地
4. 所見を growth-log.md の当日レポートに追記する（**提案のみ。nightly-review.md 自体は編集しない**）:

```markdown
### 効率化リフレクション
| タスク | 直近実行回数 | 変更あり | 変更なし | 提案 |
|---|---|---|---|---|
| TASK 1 | 8 | 2 | 6 | 対象期間を14日→7日に短縮を検討 |
| TASK 3 | 2 | 0 | 2 | 隔週（第1・3水曜）に変更を検討 |
...
提案なし: TASK X, Y
```

5. 提案が 0 件なら「効率化リフレクション: 改善点なし」と出力してスキップする

## 実行ルール

- ファイルの読み書きには Read/Write/Edit ツールを使う
- 確認や質問は一切不要。自律的に判断して実行する
- 変更は最小限・安全優先。不確かな場合は変更しない
- 全タスク完了後、変更したファイルの一覧を日本語で出力する
